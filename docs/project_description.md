# Описание проекта Mini Redis Server

## Назначение проекта

Mini Redis Server — учебный проект, в котором реализовано простое key-value хранилище на Python. Проект создан для демонстрации принципов клиент-серверной архитектуры, работы TCP-сокетов, обработки текстовых команд и хранения данных в оперативной памяти.

Проект не является полной заменой Redis. Его задача — показать базовую идею in-memory хранилища и сделать ее понятной для изучения в рамках проектной практики.

## Цель проекта

Цель проекта — разработать минимальный сервер, который принимает команды от клиентов, выполняет операции с данными и возвращает результат. Дополнительно к серверу добавлены CLI-клиент, HTTP API и демонстрационный сайт.

## Основные компоненты

- `src/server.py` — TCP-сервер Mini Redis Server.
- `src/client.py` — консольный клиент для ручной проверки сервера.
- `src/api.py` — HTTP API, которое связывает браузерную страницу с TCP-сервером.
- `site/` — статический сайт проекта и страница демонстрации.
- `docs/` — документация по проекту.
- `reports/` — материалы отчета по практике.
- `task/` — описание задания и вариативной части.

## Поддерживаемые команды

Сервер поддерживает базовые и служебные команды:

- `SET key value` — сохранить значение по ключу.
- `GET key` — получить значение по ключу.
- `DELETE key` — удалить ключ.
- `EXISTS key` — проверить, существует ли ключ.
- `PING` — проверить доступность сервера.
- `KEYS` — вывести список всех ключей или `EMPTY`, если ключей нет.
- `FLUSH` — очистить хранилище.
- `UPDATE key value` — обновить значение существующего ключа.
- `LOG` — показать последние записи журнала команд.

## Журнал команд

Сервер ведет журнал выполненных команд в оперативной памяти. Для каждой записи сохраняются время выполнения, текст команды и результат. Команда `LOG` возвращает последние записи журнала.

Журнал используется только во время работы сервера и не сохраняется после перезапуска.

## Хранение данных

Данные хранятся в оперативной памяти в словаре Python. Такой подход удобен для учебного проекта, потому что он простой и наглядный. При остановке сервера данные удаляются.

## Демонстрация

Для демонстрации используется страница `site/demo.html`. Пользователь вводит ключ и значение, нажимает кнопку нужной команды, а JavaScript отправляет HTTP-запрос в `src/api.py`. API пересылает команду в TCP-сервер и возвращает результат на страницу.

## Ограничения

- Данные не сохраняются на диск.
- Нет авторизации и разграничения доступа.
- Поддерживается учебный набор команд.
- Проект рассчитан на учебное использование и демонстрацию.

# Архитектура проекта

## Общая схема

Проект состоит из трех основных уровней:

1. Браузерная страница демонстрации.
2. HTTP API на Python.
3. TCP Mini Redis Server.

Схема взаимодействия:

```text
Браузерная страница demo.html
        |
        | HTTP-запрос через JavaScript fetch
        v
HTTP API на Python, src/api.py
        |
        | TCP-запрос
        v
Mini Redis Server, src/server.py
        |
        | Ответ на команду
        v
HTTP API
        |
        | JSON-ответ
        v
Браузерная страница demo.html
```

## Компоненты

### TCP-сервер

Файл `src/server.py` отвечает за хранение данных и выполнение команд. Сервер принимает текстовые команды по TCP-соединению и возвращает текстовый ответ.

Примеры команд:

```text
SET name Dmitriy
GET name
EXISTS name
DELETE name
PING
KEYS
UPDATE name Redis
LOG
```

### CLI-клиент

Файл `src/client.py` используется для проверки работы TCP-сервера без браузера. Он подключается к серверу, отправляет команду и выводит ответ.

### HTTP API

Файл `src/api.py` принимает HTTP-запросы от сайта. Он нужен, потому что браузер не может напрямую обращаться к TCP-серверу. API преобразует JSON-запрос в текстовую команду Mini Redis Server.

### Статический сайт

Папка `site` содержит HTML, CSS и JavaScript. Страница `demo.html` предоставляет форму для ввода ключа и значения, а также кнопки команд `SET`, `GET`, `DELETE`, `EXISTS`, `PING`, `KEYS`, `FLUSH`, `UPDATE`, `LOG`.

## Формат HTTP-запроса

Адрес:

```text
POST http://127.0.0.1:8080/command
```

Тело запроса:

```json
{
  "command": "SET",
  "key": "student",
  "value": "project-practice"
}
```

Ответ:

```json
{
  "ok": true,
  "command": "SET student project-practice",
  "result": "OK"
}
```

## Порты по умолчанию

- TCP-сервер: `127.0.0.1:6379`.
- HTTP API: `127.0.0.1:8080`.
- Статический сайт при локальном запуске: `127.0.0.1:8000`.

## Хранение и поток данных

Все данные хранятся в оперативной памяти TCP-сервера. HTTP API не хранит данные самостоятельно: оно только передает команды и ответы между браузером и TCP-сервером.

## Журналирование

TCP-сервер хранит журнал команд в памяти. В журнал попадает время выполнения, текст команды и результат. Команда `LOG` возвращает последние записи журнала, что позволяет увидеть историю работы сервера во время текущего запуска.

# Черновик отчета по проектной практике

## Тема

Mini Redis Server — учебная реализация простого key-value хранилища на Python.

## Участник

Кривошеин Дмитрий Алексеевич, группа 251-324.

## Цель работы

Цель работы — разработать учебный проект, демонстрирующий принципы клиент-серверной архитектуры, работы TCP-сокетов, обработки команд и хранения данных в памяти.

## Задачи

- Изучить базовые принципы работы key-value хранилищ.
- Реализовать TCP-сервер на Python.
- Реализовать CLI-клиент для проверки сервера.
- Добавить HTTP API для связи браузера с TCP-сервером.
- Создать статический сайт с описанием проекта.
- Добавить страницу демонстрации.
- Подготовить документацию и инструкцию по запуску.

## Выполненные работы

В ходе практики был создан Mini Redis Server. Сервер поддерживает команды `SET`, `GET`, `DELETE`, `EXISTS`, `PING`, `KEYS`, `FLUSH`, `UPDATE`, `LOG` и хранит данные в оперативной памяти. Для взаимодействия с сервером был реализован CLI-клиент. Для демонстрации через браузер был добавлен HTTP API, который принимает JSON-запросы и пересылает команды в TCP-сервер.

Дополнительно было добавлено ведение журнала команд. Сервер сохраняет в памяти время выполнения, текст команды и результат. Команда `LOG` позволяет вывести последние записи журнала во время текущего запуска сервера.

Также был подготовлен статический сайт проекта. На сайте размещены главная страница, описание проекта, журнал работы, полезные материалы и страница демонстрации.

## Результат

Результатом работы является проект с исходным кодом, демонстрационным сайтом и документацией. Проект можно запустить локально и проверить работу команд Mini Redis Server через CLI, HTTP API или браузерную страницу.

## Вывод

В результате выполнения проектной практики были закреплены навыки работы с Python, TCP-сокетами, HTTP API, JavaScript `fetch` и структурированием учебного проекта. Созданный проект показывает практическую реализацию простой серверной технологии и может использоваться для демонстрации на защите.

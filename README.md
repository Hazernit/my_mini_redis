## Вариативная часть задания

В качестве вариативной части задания выбрана практическая реализация технологии — создание собственного Mini Redis Server на Python.

Эта тема выбрана потому, что она позволяет на практике изучить принципы работы сетевых приложений, TCP-сокетов, клиент-серверного взаимодействия и простого хранения данных в формате key-value.

В рамках проекта реализуется учебный Mini Redis Server — минимальная версия key-value хранилища, которое принимает команды от клиента, обрабатывает их на сервере и хранит данные в оперативной памяти.

Проект поддерживает следующие функции:

- запуск TCP-сервера;
- подключение к серверу через CLI-клиент;
- выполнение команд `SET`, `GET`, `DELETE`, `EXISTS`, `PING`, `KEYS`, `FLUSH`, `UPDATE`, `LOG`;
- хранение данных в памяти;
- ведение журнала команд в памяти;
- обращение к серверу через HTTP API;
- демонстрацию работы через веб-интерфейс.

Проект связан с практической реализацией технологии, так как в нем вручную реализуются основные элементы серверного приложения: сетевое подключение, обработка команд, передача данных между клиентом и сервером, а также промежуточный HTTP API для связи браузера с TCP-сервером.

Код проекта находится в папке `src`.

Демонстрационный сайт находится в папке `site`.

Документация проекта находится в папке `docs`.

## Демонстрационный сайт

Демонстрационный сайт находится в папке `site`.

Сайт содержит:

- главную страницу;
- страницу о проекте;
- журнал работы;
- страницу ресурсов;
- страницу `demo.html` для демонстрации работы Mini Redis Server.

[Открыть демонстрацию](site/demo.html)

Браузерная страница `demo.html` отправляет HTTP-запрос через JavaScript `fetch` в `src/api.py`. HTTP API обращается к TCP Mini Redis Server, получает результат выполнения команды и возвращает его обратно на страницу.

Схема демонстрации:

```text
site/demo.html -> JavaScript fetch -> src/api.py -> TCP -> src/server.py -> ответ на страницу
```

## Инструкция по запуску

Запуск TCP-сервера:

```bash
python src/server.py
```

Запуск HTTP API:

```bash
python src/api.py
```

Запуск статического сайта:

```bash
cd site
python -m http.server 8000
```

После запуска откройте в браузере:

```text
http://127.0.0.1:8000/demo.html
```

Порядок проверки:

1. Запустить TCP-сервер.
2. Запустить HTTP API.
3. Запустить статический сайт.
4. Открыть страницу демонстрации.
5. Ввести ключ и значение.
6. Выполнить команды `SET`, `GET`, `DELETE`, `EXISTS`, `PING`, `KEYS`, `FLUSH`, `UPDATE`, `LOG`.
7. Проверить ответ сервера в блоке результата.

## Примеры команд

Проверка через CLI-клиент:

```bash
python src/client.py SET user Alice
python src/client.py GET user
python src/client.py EXISTS user
python src/client.py PING
python src/client.py UPDATE user Bob
python src/client.py KEYS
python src/client.py LOG
python src/client.py DELETE user
python src/client.py FLUSH
```

Проверка через HTTP API:

```bash
curl -X POST http://127.0.0.1:8080/command \
  -H "Content-Type: application/json" \
  -d "{\"command\":\"SET\",\"key\":\"user\",\"value\":\"Alice\"}"
```

## Размещение в Yandex Cloud

Для защиты проекта можно использовать два варианта запуска.

### Вариант 1. Локальный запуск

Простой вариант для демонстрации на защите:

```bash
python src/server.py
python src/api.py
cd site
python -m http.server 8000
```

После этого демонстрация доступна по адресу `http://127.0.0.1:8000/demo.html`.

### Вариант 2. Облачный запуск через Yandex Cloud

Статический сайт можно разместить в Yandex Object Storage:

1. Создать бакет.
2. Включить режим хостинга статического сайта.
3. Загрузить файлы из папки `site`.
4. Указать `index.html` как главную страницу.

Backend/API можно запустить на виртуальной машине Yandex Compute Cloud:

1. Создать виртуальную машину с публичным IP-адресом.
2. Установить Python 3 и Git.
3. Склонировать репозиторий.
4. Запустить TCP-сервер и HTTP API.
5. Открыть порт HTTP API в правилах Security Group.

Пример команд для виртуальной машины:

```bash
sudo apt update
sudo apt install -y python3 git
git clone <URL_РЕПОЗИТОРИЯ>
cd my_mini_redis
python3 src/server.py --host 127.0.0.1 --port 6379
MINI_REDIS_HOST=127.0.0.1 MINI_REDIS_PORT=6379 python3 src/api.py --host 0.0.0.0 --port 8080
```

Переменные окружения:

- `MINI_REDIS_HOST` — адрес TCP-сервера Mini Redis Server;
- `MINI_REDIS_PORT` — порт TCP-сервера Mini Redis Server.

Для облачного запуска нужно указать публичный адрес API в файле `site/script.js`:

```js
const API_URL = "http://<PUBLIC_VM_IP>:8080/command";
```

Ограничения MVP:

- данные хранятся в памяти и удаляются после перезапуска сервера;
- HTTP API не содержит авторизации;
- для публичного размещения желательно настроить HTTPS;
- при размещении статического сайта по HTTPS браузер может блокировать запросы к HTTP API.

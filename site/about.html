Проектная деятельность

Mini Redis Server

Учебная реализация простого key-value хранилища с TCP-сервером, CLI-клиентом, HTTP API и браузерной демонстрацией для проектной практики.

Что показывает проект

TCP-сервер

Принимает команды SET, GET, DELETE, EXISTS, PING, KEYS, FLUSH, UPDATE и LOG, хранит данные в памяти и возвращает текстовый ответ.

HTTP API

Связывает браузерную страницу с TCP-сервером, потому что JavaScript в браузере не подключается к TCP напрямую.

Веб-демо

Позволяет выполнить базовые команды через форму и сразу увидеть результат работы Mini Redis Server.

Цель проекта

Цель проекта — реализовать минимальное key-value хранилище по мотивам Redis, добавить клиентские инструменты и показать работу сервера через понятный веб-интерфейс.

Поддерживаемые команды

Базовые операции

SET сохраняет значение, GET получает значение, DELETE удаляет ключ, EXISTS проверяет наличие ключа.

Служебные команды

PING проверяет доступность сервера, KEYS показывает список ключей, FLUSH очищает хранилище.

Расширенные операции

UPDATE обновляет только существующий ключ, LOG показывает журнал последних выполненных команд.

Архитектура

src/server.py

TCP Mini Redis Server. Хранит данные в словаре Python и обрабатывает команды построчно.

src/client.py

CLI-клиент для ручной проверки сервера из терминала.

src/api.py

HTTP-прослойка для сайта. Принимает JSON, отправляет команду в TCP-сервер и возвращает JSON-ответ.

site/demo.html

Браузерная форма, которая отправляет запросы через fetch в HTTP API.

Поток данных

demo.html -> fetch -> src/api.py -> TCP socket -> src/server.py -> ответ на страницу

Такой подход нужен потому, что браузер работает с HTTP-запросами, но не подключается напрямую к произвольному TCP-серверу. Поэтому HTTP API выступает промежуточным слоем между интерфейсом сайта и Mini Redis Server.

Как попробовать проект в работе

Эта страница отправляет команды в HTTP API, а API пересылает их в TCP Mini Redis Server. Так можно проверить SET, GET, DELETE и EXISTS прямо из браузера.

Перед открытием демо запустите TCP-сервер и HTTP API: python src/server.py и python src/api.py. Статический сайт можно открыть через локальный HTTP-сервер из папки site.

Порядок проверки

  1. Запустите Mini Redis Server.
  2. Запустите HTTP API.
  3. Введите ключ и значение.
  4. Нажмите нужную команду: SET, GET, DELETE, EXISTS, PING, KEYS, FLUSH, UPDATE или LOG.
  5. Посмотрите ответ сервера справа.

Расширение проекта

В рамках расширения проекта были добавлены дополнительные команды PING, KEYS, FLUSH, UPDATE и LOG.

Команда PING позволяет проверить доступность сервера.
Команда KEYS выводит список всех сохранённых ключей.
Команда FLUSH очищает хранилище.
Команда UPDATE обновляет значение только у существующего ключа.
Команда LOG показывает журнал выполненных команд.

Также было добавлено ведение журнала команд. Сервер сохраняет информацию о выполненных командах в памяти: время выполнения, название команды и результат. Это позволяет отслеживать работу Mini Redis Server и демонстрирует базовый принцип логирования в серверных приложениях.
